# dev/reference_vibe-coding-skill

本目录用于沉淀“Vibe Coding（与 AI 结对开发）”相关的**可复用方法论与规则**，服务对象是：
- IT 产品经理：用于写清楚需求、约束与验收口径，从源头降低返工与误解
- Agent/开发者：用于在实现过程中保持专业性、准确性与可追溯性

内容分两类：
1) **本项目标准化规则（历史参考入口）**
   - `Vibe-Coding-Standard.md`：面向 PM/Agent 的标准化规则（MUST/SHOULD/NEVER）+ 模板 + 质量门禁。
   - `templates/`：可直接复制使用的任务/缺陷模板。

说明：
- 早期曾引入上游 `vibe-coding-cn` 仓库作为“方法论参考镜像”；当前已完成提炼与内化，为保持主仓库轻量与避免第三方内容维护成本，已移除该镜像目录。

与本项目强制规范的关系：
- 本项目业务/安全硬红线以 `rules.md` 与 `docs/*` 为准；
- 当需要把“方法论”落到“可执行门禁/产物格式”时，使用本目录的标准化规则与模板。
