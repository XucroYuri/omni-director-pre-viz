# UX 文案与界面状态规范（Vibe Coding 友好）

## 0. 一句话目标

让用户在每一步都知道“我现在在哪里、发生了什么、下一步是什么”。

## 1. 状态模型（统一标准）

- Idle：未触发
- Loading：执行中
- Success：完成
- Error：失败（含重试/查看详情）

## 2. 文案规范（可直接引用）

### Script（脚本）

- Empty：`“粘贴剧本后开始拆解”`
- Loading：`“拆解中…预计 1–2 分钟”`
- Success：`“已生成镜头：{count} 条”`
- Error：`“拆解失败：{error}（可重试）”`
- Empty 行为：显示“示例脚本/模板入口”。

### Asset（资产）

- Empty：`“尚未添加资产：角色/场景/道具”`
- Error：`“资产保存失败：{error}”`
- Empty 行为：提供“快速添加角色/场景/道具”入口。

### Matrix（母图）

- Disabled：`“需先完成提示词矩阵”`
- Loading：`“生成母图中…”`
- Error：`“母图生成失败：{error}（重试）”`
- Disabled 行为：展示原因（如“缺少提示词矩阵/资产绑定”）。

### Video

- Disabled：`“需先生成母图”`
- Loading：`“视频生成中…（预计 3–5 分钟）”`
- Error：`“视频失败：{error}（重试/查看详情）”`
- Loading 行为：显示“预计时长+后台执行说明”。

### Export

- Disabled：`“请先生成母图或视频”`
- Loading：`“导出中…”`
- Success：`“导出完成：{path}”`
- Error：`“导出失败：{error}（查看日志）”`
- Success 行为：提供“打开目录/复制路径”。

### 保存状态

- Unsaved：`“未保存”`
- Saving：`“保存中…”`
- Saved：`“已保存于 {time}”`
- Error：`“保存失败：{error}（重试）”`

## 3. 交互规范

- 禁用按钮必须展示原因（tooltip 或辅助文案）。
- 错误必须提供“重试/查看详情”。
- 成功提示尽量不打断流程（toast 优先，modal 次之）。
- 错误信息需包含错误码或可追溯 ID。
- 操作按钮需有最小冷却或防抖（避免重复提交）。

## 4. 状态映射建议

- Script 拆解：Script 状态 + TaskPanel 状态双通道展示。
- 母图/视频：卡片状态 + 弹窗状态同步。
- 导出：顶部 Toast + 导出面板日志。

## 5. 风险与注意

- 不允许在关键链路使用 alert/confirm。
- 长耗时操作必须有状态可见。
