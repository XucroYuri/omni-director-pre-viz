# 页面级 UX 走查清单（逐页验收要点）

> 使用方式：逐页走查并记录“通过/失败/备注”。

## 1) Renderer 主界面（脚本/拆解/配置）

- [ ] 空状态：脚本输入提示清晰
- [ ] Loading：拆解过程有明显进度/状态
- [ ] Error：失败有可重试路径
- [ ] 保存状态：未保存/保存中/已保存可见

## 2) MatrixPromptEditor（母图/视频/预览）

- [ ] 母图生成前有禁用原因提示
- [ ] 生成中显示预计时长
- [ ] 失败可重试且不丢上下文
- [ ] 视频生成状态与队列一致
- [ ] 预览弹窗关闭行为一致

## 3) Sidebar（导出/资产/全局配置）

- [ ] 资产为空提示入口明显
- [ ] 上传失败提示友好
- [ ] 导出成功/失败文案统一
- [ ] 导出失败提供诊断入口

## 4) TaskPanel（任务队列）

- [ ] 任务状态可追踪（Queued/Running/Failed/Completed）
- [ ] 失败任务可重试/取消
- [ ] 任务进度条展示正确

## 5) Web - Home 页面

- [ ] 页面结构清晰、导航入口可见
- [ ] 主要入口按钮文案一致
- [ ] 说明文字符合术语规范

## 6) Web - Episodes 列表页

- [ ] 空状态有引导
- [ ] 新建 Episode 流程清晰
- [ ] 打开 Episode 入口明显

## 7) Web - Episode 详情页

- [ ] 保存/拆解按钮状态准确
- [ ] 镜头列表显示清晰
- [ ] 母图生成与状态反馈一致
- [ ] 导出入口一致且可验证

## 8) Web - Ops Queue 页面

- [ ] Loading 文案清晰
- [ ] 错误提示可行动（重试/诊断）
- [ ] 空状态清晰
- [ ] 操作反馈不依赖 alert
